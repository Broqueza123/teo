1. CSS Position Property

The position property in CSS specifies how an element is positioned in a document. There are five different position values:

  • static (default)
  • relative
  • absolute
  • fixed
  • sticky

Position: Static (Default)

Elements are positioned according to the normal flow of the document. The top, right, bottom, and left properties have no effect.

.static-box {
    position: static;
    /* top, right, bottom, left properties have no effect */
}
Static

Position: Relative

Elements are positioned relative to their normal position. Setting top, right, bottom, and left properties will cause it to be adjusted away from its normal position.

.relative-box {
    position: relative;
    top: 20px;
    left: 20px;
}

Position: Absolute

Elements are positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed). If an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.

.absolute-box {
    position: absolute;
    top: 50px;
    right: 50px;
}
Absolute
50px from top
50px from right

Absolute Positioning with a Positioned Parent

When an absolute element has a parent with a position other than static, it is positioned relative to that parent:

.reference-element {
    position: relative;
}

.absolute-child {
    position: absolute;
    bottom: 10px;
    right: 10px;
}
Positioned Parent (position: relative)
Absolute Child

Position: Fixed

Elements are positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element.

.fixed-box {
    position: fixed;
    bottom: 20px;
    right: 20px;
}
Note: Look at the bottom right corner of the screen. There's a fixed positioned orange box that stays in place as you scroll.

Position: Sticky

Elements are positioned based on the user's scroll position. A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed).

.sticky-box {
    position: sticky;
    top: 20px;
}
Note: Scroll down to see the sticky box in action. It will remain at 20px from the top once you scroll past its original position.

Summary of Position Properties

  • static: Default positioning according to the normal document flow
  • relative: Positioned relative to its normal position, creating a positioning context for child elements
  • absolute: Positioned relative to the nearest positioned ancestor
  • fixed: Positioned relative to the viewport, stays in place when scrolling
  • sticky: Positioned based on scroll position, toggles between relative and fixed
Important: When using positioning, remember that the top, right, bottom, and left properties work differently based on which position value you choose.

6. CSS Transitions

CSS transitions allow you to change property values smoothly, over a given duration.

How to Use CSS Transitions?

To create a transition effect, you must specify two things:

  • The CSS property you want to add an effect to
  • The duration of the effect

Basic Transition Example

Hover over the red box to see a width transition:

div {
    width: 100px;
    height: 100px;
    background: red;
    transition: width 2s;
}

div:hover {
    width: 300px;
}

Multiple Properties Transition

Hover over the blue box to see width and height transitions with different durations:

div {
    width: 100px;
    height: 100px;
    background: blue;
    transition: width 2s, height 4s;
}

div:hover {
    width: 300px;
    height: 200px;
}

Transition Properties

In this chapter you will learn about the following properties:

  • transition - Shorthand for all transition properties
  • transition-delay - Specifies a delay for the transition effect
  • transition-duration - Specifies how many seconds a transition takes
  • transition-property - Specifies which properties to transition
  • transition-timing-function - Specifies the speed curve of the transition

7. CSS Animations

CSS allows animation of HTML elements without using JavaScript!

What are CSS Animations?

An animation lets an element gradually change from one style to another. You can change as many CSS properties you want, as many times as you want.

The @keyframes Rule

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style at certain times.

Basic Animation Example

This changes the background-color from red to yellow over 4 seconds:

@keyframes example {
    from {background-color: red;}
    to {background-color: yellow;}
}

div {
    animation-name: example;
    animation-duration: 4s;
}

Multiple Keyframes Example

This changes both background-color and shape at different stages:

@keyframes color-shape {
    0% {background-color: red; border-radius: 0;}
    50% {background-color: blue; border-radius: 50%;}
    100% {background-color: green; border-radius: 0;}
}

div {
    animation-name: color-shape;
    animation-duration: 4s;
}

Animation Properties

In this chapter you will learn about the following properties:

  • @keyframes - Specifies the animation code
  • animation-name - Specifies the name of the @keyframes animation
  • animation-duration - Specifies how long an animation should take
  • animation-delay - Specifies a delay before the animation starts
  • animation-iteration-count - Specifies how many times an animation should run
  • animation-direction - Specifies whether to play in reverse on alternate cycles
  • animation-timing-function - Specifies the speed curve of the animation
  • animation-fill-mode - Specifies what values are applied before/after animation
  • animation - Shorthand for all the animation properties
